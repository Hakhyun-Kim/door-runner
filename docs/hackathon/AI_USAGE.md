# 두 문 러너 (Door Runner) — AI 활용 기술 문서

> NAN 2026 (NHN Game × AI Hackathon) 사전과제 제출물 4
> 제출자: 김학현 (개인 참여)

## 1. 사용한 AI 도구

| 도구 | 용도 |
|------|------|
| **Claude Code** (Anthropic) | 주력 개발 도구. Claude 모델 기반 CLI 코딩 에이전트로, 기획 정리·코드 작성·디버깅·문제 은행 생성·빌드 설정·문서 작성까지 개발 전 과정에 사용 |

이미지 생성 AI, 사운드 생성 AI는 사용하지 않았습니다 — 그래픽은 코드(절차 생성 지오메트리·캔버스 드로잉), 사운드는 Web Audio 합성으로 해결했으며 이 코드들 역시 Claude Code로 작성했습니다.

## 2. AI 활용 구조 (개발 방법론)

### 2.1 CLAUDE.md — 지속 컨텍스트 문서
프로젝트 루트의 `CLAUDE.md`에 게임 규칙·아키텍처·코딩 원칙·로드맵을 유지하고, Claude Code가 매 세션 이를 읽고 일관된 맥락에서 작업하도록 했습니다. AI와의 협업에서 "매번 다시 설명"하는 비용을 없애는 핵심 장치였습니다. 주요 수록 내용:

- phase 상태 머신 정의: `run`(달리기) → `ask`(문 선택) → `pass`/`crash` → `over`
- 좌표계 규칙 같은 실수 방지 메모 (예: "카메라가 +z를 보므로 화면 왼쪽 = +x")
- UI 원칙: "문구는 모두 한국어, 초등학생 눈높이, 오답이어도 기죽지 않는 부드러운 문구"

### 2.2 대화형 반복 개발 루프
사람이 방향·검수를 맡고 AI가 구현을 맡는 분업으로 진행했습니다.

1. 사람: 기능/버그를 한국어로 서술 (스크린샷·현상 포함)
2. Claude Code: 관련 파일 탐색 → 수정 → `npm run typecheck` 자동 실행
3. Claude Code: 개발 서버 브라우저 프리뷰로 실제 동작 확인 (숨김 탭에서 rAF가 멈춰 3D가 안 그려지는 문제는 `?rafshim` 쿼리 심을 만들어 우회 — 이 검증 장치 자체도 AI가 제안·구현)
4. 사람: 실기기(스마트폰·PC)에서 조작감·가독성 최종 검수

### 2.3 원본 게임의 이식 (Roblox → 웹 3D)
이 게임은 동일 기획의 로블록스판(Luau, 동일 제작자)을 먼저 만들었고, 그 소스를 Claude Code에 참조시켜 트랙 치수·색·연출 비율을 three.js로 재현했습니다. "다른 엔진의 기존 코드를 스펙 문서처럼 사용"하는 AI 활용 방식입니다.

## 3. 주요 프롬프트 및 지시 사항

실제 개발 중 사용한 지시를 영역별로 요약한 대표 예시입니다.

### 3.1 문제 은행 생성 (240문제)
> "초등 수학 문제 은행을 만들어줘. 1-1부터 6-2까지 학기당 20문제. 한국 초등 수학 교육과정의 학기별 단원 범위를 반드시 지켜 — 예: 2-2는 곱셈구구, 4-1은 각도와 세자리×두자리 곱셈, 6-2는 분수÷분수·비례식. 형식은 `{q: 문제, a: 정답, w: 오답}`이고 오답은 학생이 실제로 헷갈릴 만한 값으로."

생성 후 사람이 전 문항의 정답·오답을 검수했고, 학기 범위를 벗어난 문항은 지적하여 AI가 교체했습니다.

### 3.2 3D 씬 구현 (로블록스판 재현)
> "로블록스판 소스를 참고해서 three.js로 같은 느낌의 트랙을 만들어줘. 구간 길이 38, 문 7×11, 보라 게이트, 유리 옆벽, 교대 바닥 색까지 비율 그대로. 캐릭터가 +z로 달리고 카메라가 뒤에서 추적."

> "ask 구간에서 버튼으로 고르는 게 아니라, 로블록스처럼 누르는 방향의 문으로 캐릭터가 걸어가서 몸이 닿으면 판정되게 바꿔줘 (Touched 감각). 문 사이 벽은 막혀서 못 지나가야 해."

### 3.3 게임 로직·연출 디테일
> "run/pass 구간 끝을 타이머로 잡지 말고 씬 쪽 콜백(onArrive/onPassed)이 알려주게 해줘 — 기기 성능에 따라 어긋나는 문제가 있어."

> "오답으로 💥 난 직후 누르던 손가락 때문에 연속 충돌이 나. 오답 시 누름을 강제 해제해줘."

> "세로 화면에서 두 문이 다 안 보여. 화면 비율에 따라 FOV를 자동 보정해줘."

### 3.4 에셋 없는 에셋 파이프라인
> "한글 문제 텍스트를 3D 문 위에 그려야 해. 폰트 파일 없이 캔버스에 그려 텍스처로 만들어줘. 영역에 맞춰 자동 줄바꿈·축소되게. (로블록스 SurfaceGui 방식)"

> "효과음 파일 없이 Web Audio로 정답 아르페지오, 오답 저음을 합성해줘."

### 3.5 배포 파이프라인
> "NHN 해커톤 사전과제 제출 요건을 보고 뭐부터 해야 할지 정리하고, 필요한 문서를 만들어줘. GitHub Pages로 심사자가 링크 클릭만으로 플레이할 수 있게 해줘."

Claude Code가 vite base 설정, GitHub Actions 워크플로 작성, GitHub API를 통한 저장소·Pages 설정, 배포 후 HTTP 검증까지 수행했고, 이 제출 문서 초안도 작성했습니다.

### 3.6 지시 사항의 공통 원칙
- 결과물 기준(초등학생 눈높이 문구, 로블록스판 비율 재현 등)을 프롬프트에 명시하고, 구현 방법은 AI에 위임
- 버그는 현상 그대로 서술 ("연속 충돌이 난다") — 원인 추정은 AI가 수행
- 반복될 규칙은 프롬프트가 아니라 CLAUDE.md에 승격하여 영구화

## 4. AI 산출물 검증 방법

| 대상 | 검증 |
|------|------|
| 수학 문제 240문항 | 사람이 전수 검수 (정답 오류·학기 범위 이탈 교정) |
| 코드 | `tsc --noEmit` 타입 검사 + Claude Code의 브라우저 프리뷰 자가 검증 + 실기기 테스트 |
| UI 문구 | 초등학생 눈높이·부드러운 어조 기준으로 사람이 검토 |
